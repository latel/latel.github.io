为什么每个前端工作者都应该有一台 MacBook

发表于 2014-12-11   更新于 2016-07-03   |   分类于 程序员那些事   |  

极致的工程学美感

首先不可否认的是，很多人买Macbook，都是被它漂亮的外形所吸引，在各大PC厂商忙着推出超薄本之前，Macbook的外形基本是独步天下，不过昂贵的售价也是让很多人可望不可即。当年作为穷学生的我，也是很期望自己有这么一台漂亮的电脑的 ：）。

工作之后，慢慢的有了一些积蓄，终于体会到有钱的好处。自己的那台用了快5年的Acer 4736ZG也是不堪重负，各种问题了，于是狠狠心，在14年买了台中配的Macbook Air，当时犹豫了很久，到底是买 retina Pro还是Air，待机续航时间都差不多，最后还是Air更加俊俏的外形和重量吸引了我，配了个8G内存以备虚拟机之用，买回来发现，作为一个前端从业者，是大大的够用了。

漂亮的不只是外形，操作系统设计的也是赏心悦目，当然，我指的是 OSX 10.9 Mavericks，我比较偏爱拟物风格。10.10及之后的系统基本上就是渣，界面丑死了，还无穷多的BUG，无穷多的不稳定。

工作效率的提升

OSX的系统设计的的确是非常的好用！

  • 1) 到处都是无穷多的快捷键：刚开始可能不适应无处不在的超多的快捷键，但是一旦你掌握了规律，举一反三，就会发现快捷键对工作效率的提升简直是质的飞跃，相反在Windows下面我就基本上没怎么使用快捷键（除了剪切，粘贴和保存这三个），还是系统的设计对大家的日常操作有影响；

  • 2) 设计优良的效率管理工具和软件：刚刚使用OSX你可能会非常不习惯，那些在Windows下玩的很顺手的软件很多时候都找不到对应的工具，但是慢慢的你会发现很多同样没有Windows版本的软件，这些软件不仅界面设计考究，功能更是无出其它，尤其是设计方面的软件入zeplin、sketch等等等等。你会体会到最棒的流程绘制软件omnigraffle等，当然，PS这些最为依赖的软件到了OSX平台上依然可以使用。

  • 3）安静：由于使用的是铝合金一体成型，所以Macbook的散热非常好，如果不是看视频或者在网页上看flash视频的话，大多数时候，Macbook的风扇是不转的，非常的安静。不会打断你高度集中超速运行的大脑。

  • 4) 不用关机：由于使用的是固态硬盘，不存在划伤硬盘盘面的问题，OSX是不需要关机的。当你临时有事的时候，只需要轻轻的合上屏幕，带走，然后在需要继续工作的时候立刻继续你的工作。我经常一个月才真正意义上的重启电脑一次。关机是什么？ Sorry， I beg your pardon?

Unix的优雅哲学

开发人员基本上都更多的时间在Linux环境下工作，因为强大的终端和各种经久历史所积累下来好用的终端程序，开发者可以高效的用命令行来完成他们的工作。OSX是源于UNIX的，和Linux血缘相近，Linux有的，OSX基本上都有，暂时没有的，未来也可以有。最为重要的是，对于前端从业者来说，OSX环境更加友好，这也是吸引我买Macbook的最重要的原因之一。

一些日常使用的App程序和吸引我的原因（持续更新）：

应用程序 描述
Mou Mou 是一个撰写 markdown 语法文档的编辑器。是个人觉得的目前 Macosx 环境下最好用的 md 文件编辑器，简洁的界面，强大的功能，另外再配置里打开按键音，那感受，有种在充满油墨香气的复古打印机上写小说的舒服感受。PS，这个软件还是国产的哦，不过听说后来作者打算转让该软件的所有权，也不知道后面怎么发展的。
Macdown Mou 这款软件更新实在太慢，很多地方都已经显得很过时，目前我更爱它的开源实现，另一款叫做 Macdown 的 markdown 编辑器，推荐使用。
Transmit 和 Windows 下的 Flashfxp 齐名，不仅是免费软件，还非常好用，优雅的界面设计你值得拥有。
Sequel Pro Mysql 服务器管理软件，你可以认为是 GUI 版本的 Phpmyadmin，谁功能更加强大我不好说，但是单从用户易用性来说，大家都知道。
Dash 不知道某个函数怎么调用，还忙着再百度上或是官方文档上到处搜索，如果离线没有网络了怎么办？Dash正是为此而生，你可以通过它预先下载很多的（超多的）语言的手册并且会自动索引。这样你只需再搜索栏输入你要查找的内容，他就会轻而易举帮你找到你需要的内容和位置。
Reeder Google Reader已死，有事烧纸。使用过网上很多所谓的 Google Reader 的继任者，可是总感觉哪里不对。要么是被墙使用极其不便，要么是欠缺一些自己需要的体验。Reeder是一款本地的 Rss 阅读器客户端，其良好的设计和操作体验，一定会令你折服的，尤其是全屏阅读，再在暖暖的冬日阳光下泡杯温温的咖啡，那体验！遗憾的是， Reeder是一个付费软件。
iKana 一个学习日语的小工具， おはよ　　秋の雨，如果你需要一款日语输入法，谷歌日语输入法不会让你失望的。
iTerm2 iTerm2 是一个虚拟终端程序，继承了 iTerm1 的优秀体魄，好像在 Linux 下也听到过这个名字，不过当时一般都是用 DE 自带的终端程序。不过发现 Macosx 自带的总感觉不太好用了，至少没 iTerm2 好用。
DayOne 一款日记应用，功能说多不多，说少不少，恰好能满足记日记的需求。也许这就是 mac 下简洁的设计原则吧。
Atom 一款由 Github 基于前端技术链打造的开源面向未来的编辑器，非常适合前端同学捣鼓，报系统体验灰常的棒，插件体系非常友好，现已作为我日常开发的主力编辑器。
Visual Studo Code 慢慢的发现 Atom 虽然对前端很友好，但是性能不是一般的差。Visual Studio Code是 M$ 公司专门从 Visual Studio 分了一拨人来虐狗的。打开大文件性能极高，一般的编辑器打开超大文件时都会卡死半天。而且也支持插件，虽然插件系统还是有些不好用，但是该有的都有，最重要的是，也是前端友好的编辑器。
MacVim Vim 学会了不用总觉得浪费，这个 MacVim 是 VIM 在 OSX 下的一个实现，因为很轻巧，启动速度快，非常适合临时查看和修改一下下源码使用，日常还是在 Atom 里装了一个叫 Vim-mode 的包来模拟极少部分的 Vim 使用习惯（基本就是移动光标……）。
PicGIF 这是一个可以把图片或者视频制作成 gif 的图片的图形化工具，我用它来把我和女票生活中的各种欢乐场景制作成表情，在微信和QQ中使用 :-)
Office 2016 自然我是推崇苹果自带的办公套件的，但是文档这东西并不是只给自己使用的，还要穿越，所以标准的微软 Office 套件还是必不可少的。不过 2016 只支持 OSX 10.10+，其实我对 OSX 10.10+ 系统无爱，还是喜欢 10.9 版本，不过自从 MacBook Air 被偷了后，再买就只能买到最低预装 Yosemite 版本的了，哎，人，还是向前看吧。如果你只想做一个演讲稿，而且不需要和别人分享的话，使用 【苹果的 Keynote 吧，绝对会让你眼前一亮，乔布斯就是用这个软件做 WWC 演讲的。
CheetSheet 看名字一定会理解错，这可不是一款游戏作弊软件，她的作用是在任何软件界面中按住 CMD 2秒钟，会浮现一个弹层，告诉你这个软件有哪些快捷键可以用，这年头软件这么多，记住每一个快捷键是很困难的事情，所以有备无患吧。不过对我这个没事喜欢一直按着 CMD 的人来说，还是有点伤的。
Shazam 这款软件会告诉你电脑里和身边播放的音乐的名字，音乐爱好者利器，而且他是静默的，会一直在后台默默的为你工作。当你做一些邪恶的事情时，要注意一下 :-)
iStat Menus 这款软件类似于 Linux 下面板程序上那个检测 CPU 和 内存的小插件，不过界面可是美多了。而且功能更加丰富。程序员估计都会喜欢这玩意儿。
The Unarchiver 系统自带的解压程序只能解压 zip 等开源压缩格式，所以你一定会需要一个免费的解压软件。而且使用体验超级棒，从 Windows 来的你，一定会喜欢的。
QQ音乐，网易云音乐和虾米音乐 读文章和写代码时听音乐是再惬意不过的事情。现在国内版权化越来越严厉，基本分为阿里系和腾讯系。所以两边的音乐都要有。网易云音乐是新起之秀，最近大家都很喜欢，QQ音乐体验也不错，就是阿里系的虾米音乐使用体验实在是够烂的，但是虾米的推荐系统实在叼的不行，而且很多歌只有阿里系的虾米有，比如大部分日文和韩文歌，所以不得不装一个。这些都是联网的音乐播放器，如果你是纯粹的音乐爱好者，我觉得使用系统自带的 iTunes 就已经非常的到位了。
VMWare Fusion 很多实用，还是需要去 Windows 下做一些工作，所以此处应该有一个虚拟机。是的，我不推荐使用 Parallels Desktop，我觉得它徒有虚名，估计就是广告做的好点，用起来体验并没有觉得有多好，而且用到后来超级卡，相比 VMWare 就一直很稳定，速度也很快。
OmniGraffle 最好用的流程绘制工具

一些命令行工具

应用程序 描述
brew 相当于没有图形界面的 App Store，或是 Linux 下的 yum，apt 和 pacman. 主要是面向开源和自由软件，可以通过它安装很多 Linux 下的套件。

总之

OSX 给我带来的工作便利性和节省的时间是无法估量的。

当然我还是很喜爱 Archlinux 的设计思想的，还是需要在虚拟机里装上一个以备使用，借助 OSX 极其便捷的虚拟桌面切换能力，一切都感到那么的自然和亲切。
